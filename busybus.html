BusyBus Case Study

View GitHub
  • Summary:

    BusyBus is a mobile app to assist users in navigating city transit.

  • Role:

    User Researcher, Interaction Designer, Visual Designer, Information Architect, Usability Researcher, Front End Developer

  • Tools:

    Figma, Google Survey, Atom, GitHub

  • Deliverables:

    Usability Test Summary, Sketches, Low-Fidelity Wireframes, High-Fidelity Mock-Up, Market Research, Personas, User Stories

The Problem

Thousands of people rely on public transit. To accomodate, numerous bus routes have been added, many buses stop at the same stop. The goal is to help riders know when and which bus is arriving.

The Solution

BusyBus works to resolve issues by tracking buses, alerting issues and allowing alternate routes. The wireframe show this based on bus stop location.

User Reasearch Takeaways

Users don't feel completely confident in using bus transporation. They would like clearer navigation information such as knowing route options, bus issues, next buses and visual information.

Wireframes

Paper sketch layout of 3 screens, which show ways to search buses, bus location and different times a bus will arrive.

Paper sketched cleaned up in digital form. An added feature is a photo of the bus stop to encourage user confidence.

Prototypes

Iteration from wireframe to prototype with images and color. The final round was coded to GitHub.
